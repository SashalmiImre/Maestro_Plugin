## Mit csináltam

**A.1 — Adatmodell és taxonómia ([[Döntések/0008-permission-system-and-workflow-driven-groups|ADR 0008]] séma-szint)** — a négy nyitott A.1 alpont (`A.1.3`, `A.1.4`, `A.1.5`, `A.1.9`) teljes implementálása. Codex előzetes és záró review pass-szal.

### A.1.3 + A.1.4 — `permissionSets` + `groupPermissionSets` collection
- Új CF action: `bootstrap_permission_sets_schema` az [invite-to-organization main.js](packages/maestro-server/functions/invite-to-organization/src/main.js)-ben (a `bootstrap_publication_schema` mintáját követi: owner-only, idempotens, `created` / `skipped` / `indexesPending` listák).
- Két új collection idempotens létrehozással: üres collection-szintű perms + `documentSecurity: true` (a doc-szintű team ACL ad olvasási jogot az ADR 0003 minta szerint).
- `permissionSets` attribútumok: `name`, `slug`, `description`, `permissions[]`, `editorialOfficeId`, `organizationId`, `archivedAt`, `createdByUserId`. Indexek: `office_slug_unique`, `office_idx`, `org_idx`.
- `groupPermissionSets` attribútumok: `groupId`, `permissionSetId`, `editorialOfficeId`, `organizationId`. Indexek: `group_set_unique`, `office_idx`, `group_idx`, `set_idx`.
- Két új env var (`PERMISSION_SETS_COLLECTION_ID`, `GROUP_PERMISSION_SETS_COLLECTION_ID`) — action-szintű guard, a többi action működése érintetlen.

### A.1.5 — `compiled.requiredGroupSlugs[]`
- A workflow `compiled` JSON új top-level mezője a hivatkozott felhasználó-csoport slug-ok kanonikus listája `{slug, label, description, color, isContributorGroup, isLeaderGroup}` objektumokkal.
- A [compiler.js](packages/maestro-dashboard/src/features/workflowDesigner/compiler.js) `compiledToGraph` és `graphToCompiled` bővítve. Új helperek: `collectAllReferencedSlugs`, `reconstructRequiredGroupSlugs`, `normalizeRequiredGroupSlugs`.
- A `compiled.contributorGroups[]` és `leaderGroups[]` mostantól **autogeneráltak** a `requiredGroupSlugs[].isContributorGroup` / `isLeaderGroup` flag-ekből — a kimenet konzisztens, a UI még a régi mezőket szerkesztheti A.4.6-ig.
- Read-time fallback rekonstrukció a legacy compiled (requiredGroupSlugs nélküli) doc-okra — NEM DB write, semmi migrációs hatás.
- `defaultWorkflow.json` (mindkét példány: shared + server inline) frissítve a 7 alapértelmezett slug `requiredGroupSlugs[]` blokkjával.

### A.1.9 — Hard contract validátor
- Új shared modul: [compiledValidator.js](packages/maestro-shared/compiledValidator.js).
- `validateCompiledSlugs(compiled)` 8 ellenőrzéssel: `requiredGroupSlugs` duplikáció + 7 slug-hivatkozó mező (`transitions`, `commands`, `elementPermissions`, `leaderGroups`, `statePermissions`, `contributorGroups`, `capabilities`).
- Error code-ok: `unknown_group_slug`, `duplicate_required_group_slug`, `invalid_required_group_entry`, `invalid_compiled`. UI-friendly összefoglaló: `summarizeValidationErrors()`.
- Becsatornázás a [WorkflowDesignerPage.jsx](packages/maestro-dashboard/src/features/workflowDesigner/WorkflowDesignerPage.jsx) `handleSave` flow-jába a meglévő graph-validátor (`validateWorkflow`) UTÁN, a state-removal check ELŐTT.
- A.2.1 (CF write-path enforcement) ugyanezt a shared modult fogja használni — kliens-szerver konzisztens.

## Mi volt nehéz / blokkoló

- **Default workflow két példányban**: a [packages/maestro-shared/defaultWorkflow.json](packages/maestro-shared/defaultWorkflow.json) és a [packages/maestro-server/functions/invite-to-organization/src/defaultWorkflow.json](packages/maestro-server/functions/invite-to-organization/src/defaultWorkflow.json) byte-azonosak (server-CF inline másolat, hogy a CF-en belül require-olható legyen). Codex jelezte előre — mindkettőt frissítettem.
- **Legacy UI vs. autogenerálás**: a `WorkflowPropertiesEditor` még a régi `metadata.leaderGroups` mezőt szerkeszti. A `graphToCompiled` rekonstrukciós ága hidat ad — a régi UI input bekerül a `requiredGroupSlugs[]`-be `isLeaderGroup: true` flag-gel, és onnan autogenerálódik a `leaderGroups[]`. Stabil az átmenet az A.4.6 (UI átállítás `requiredGroupSlugs[]` flag-szerkesztésre) idejéig.

## Új ismeret / felfedezés

- Az Appwrite `databases.createCollection(databaseId, id, name, permissions=[], documentSecurity=true, enabled=true)` API üres `permissions` + `documentSecurity: true` flaggel hozza létre a collection-t — ez a kívánatos minta a tenant-scope ACL-hez. A Console-on viszont továbbra is meg kell győződni, hogy a `rowSecurity` flag aktív (ADR 0003 deploy checklist).
- A `requiredGroupSlugs[]` autogenerált `contributorGroups[]` és `leaderGroups[]` szétbontása lehetővé teszi, hogy a workflow Designer egyetlen szerkesztési ponton kezelje a slug-listát (A.4.6) — nincs többé "két forrás, manuális szinkron" anti-pattern.

## Harden pass (késő este)

A frissen implementált A.1 kódra `/harden` skill — 6 Codex review iter (baseline + adversarial + 4 verif) + simplify + 6 stop-hook regresszió-fix.

**Javított (10):**
1. `compiledValidator.js` 3 helyen exception-throw nem-array input-ra → `asSlugArray`/`asObject` helperek + `requiredGroupSlugs` típus-check.
2. `bootstrap_permission_sets_schema` `createIndex` 400 elnyelés szigorítás (`||` → `&&`).
3. `ImportDialog` pre-import warning (hard contract).
4. `normalizeAndValidateImport` pre-normalize raw validation — anti-mask a típushibák ellen.
5. Legacy compiled (no `requiredGroupSlugs`) skip a raw validációból — meglévő export-ok loadability fix.
6. `ImportDialog` strict block (button disabled + handler guard) — corrupt graph nem kerül a Designer state-be.
7. `ImportDialog` + `CreateWorkflowModal` stale `importData` reset.
8. Out-of-order parse completion guard (`parseSeqRef`).
9. `CreateWorkflowModal` submit-vs-parse race (`isParsingImport` state).
10. Close/unmount-time parse seq invalidation.

**Simplify (5):** `pushUnknown` helper, `normalizeAndValidateImport` shared, ImportDialog normalize cache, `ensure*` lokális helperek, `applyImport` dead code.

**Elutasított (6 NOISE/scope):** `console.error` (CLAUDE.md megengedi), `WorkflowDesignerPage:386` legacy mező (A.4.6 átmenet), cross-action `requireOrgOwner` (A.1 scope-on kívül), Promise.all CF parallel (rate-limit), cross-module slug iterátor (refactor drága), backend `update_workflow` slug guard (A.2.1 scope, user OK'd).

**Rabbit-hole stop (user-döntés iter 6 után):** unmount-after-setState warning-ok (`closeModal()` race + `handleSubmit` finally) — React 18+ csak warning, nem corrupt; nem real-world bug.
