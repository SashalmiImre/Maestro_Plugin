# 2026-05-05

## Mit csináltam

**B.4 — Workflow Extensions Phase 0 Plugin runtime teljes implementáció** ([[Döntések/0007-workflow-extensions|ADR 0007]] Phase 0). A `feature/maestro-redesign` branchen, 8 fájl módosítás + 1 új fájl, ~470 sor új kód, Codex tervi + záró + 2× follow-up review CLEAN.

### Implementációs összefoglaló

- **B.4.1 + B.4.4 — `extensionRegistry.js`** (új modul, ~470 sor): `buildExtensionRegistry(snapshot)` JSON parse + shape-validate → `Map<slug, {name, kind, scope, code}>`. Kétszintű fail-closed: top-level (parse/nem-objekt → üres Map) ÉS per-entry (hibás entry skip + logWarn, többi entry betöltődik) — JSDoc-ban explicit dokumentálva (Codex follow-up Medium fix). `resolveExtension` strukturált error-okkal (`unknown_slug` / `kind_mismatch` / `no_registry`). `executeExtensionScript(code, input)` — hex-encoded JSON input + ExtendScript sablon kézi JSON-szerializerrel **host-szintű `{ ok, value | error }` envelope**-ot ad: a user-kód SOSEM érintkezik a wrapperrel, így nincs `__ext_error` sentinel-collision (Codex tervi review Critical fix). Az input-parse `JSON.parse`, NEM `eval` (Codex High fix). Try/catch minden ágon: `input_parse`, `load_error`, `exec_error`, `missing_maestro_extension_function`, `doScript_failed`, `parse_envelope_failed`. `dispatchExtensionValidator` / `dispatchExtensionCommand` strict shape-check (`_checkValidatorShape` / `_checkCommandShape`) — kontraktus-szegő válasz (nem-string `errors[]`, nem-bool `isValid`/`success`, nem-string `error`/`message`) → explicit `[ext.<slug>] invalid_output_shape: ...` (Codex follow-up Medium fix).

- **B.4.2 — Engine + runner + commands integráció**:
  - [[Komponensek/Workflow Engine]] `validateTransition`/`executeTransition` 5./6. paraméterre `extensionRegistry = null` (visszafelé kompatibilis, default-szal).
  - [[Komponensek/Validation Runner]] `validate(...)` `ctx.extensions` mezőn propagálja a registry-t.
  - `StateComplianceValidator.validate(context)` switch-be `default` ág: `isExtensionRef(validatorName)` → `_checkExtensionValidator(slug, article, ctx.extensions, results)` → `dispatchExtensionValidator()`.
  - `core/commands/index.js` `executeCommand` korai `isExtensionRef(commandId)` ág → `dispatchExtensionCommand(context.extensions, ref.slug, { article: context.item, publicationRoot: context.publication?.rootPath ?? null })`. A built-in `COMMAND_REGISTRY` ágat NEM érinti (cross-cutting nélküli routing).
  - **UI bekötés**: `ArticleProperties.jsx` + `PropertiesPanel.jsx` `useMemo(() => buildExtensionRegistry(activePublication?.compiledExtensionSnapshot), [$id, snapshot])` — snapshot-preferáló identitás-deps. ### Codex review-k

1. **Tervi review** (CLEAN + Critical/High fix): a `__ext_error` sentinel ütközéses → cseréje `{ ok, value | error }` envelope-ra; `eval(payload)` helyett `JSON.parse`. Mindkettőt Phase 0-ban az implementáció előtt beépítettem.
2. **Záró review** (Medium 2× → BLOCKING):
   - JSDoc a `buildExtensionRegistry`-ben pontatlan ("üres Map" vs. tényleges per-entry skip) → JSDoc frissítés két szint elkülönítésére.
   - Output normalizálás engedékenység (`{isValid:false, errors:[]}` és `{success:false}` `error` nélkül diagnosztika nélkül átengedett) → explicit `invalid_output_shape: ... empty` üzenet pótlás.
3. **Follow-up review** (BLOCKING marad): a validator `errors:[{code:'X'}]` (nem-string elemek) `isValid:true` mellett még csendben átment. → Strict shape-check helper-pár (`_checkValidatorShape`/`_checkCommandShape`) bevezetése, a normalizálás ELŐTT futtatva.
4. **Final pass review** (CLEAN): a strict shape-check lefedi az összes előző Medium és Low finding-et.

### Snapshot-immune szemantika

Az aktivált publikáció `compiledExtensionSnapshot`-ja a runtime kanonikus forrása. Az extension-doc Realtime mutáció (rename, code-szerkesztés, archive) NEM érinti az élő publikáció validálását vagy parancs-futtatását. Új aktiváció után új snapshot kerül be (a `activate_publication` action a workflow `compiled` JSON-ból frissen scan-eli az `ext.<slug>` hivatkozásokat és a hozzájuk tartozó extension-kódokat).

### Megmaradt (B blokk)
