### Codex review-k

1. **Tervi review** (CLEAN + Critical/High fix): a `__ext_error` sentinel ütközéses → cseréje `{ ok, value | error }` envelope-ra; `eval(payload)` helyett `JSON.parse`. Mindkettőt Phase 0-ban az implementáció előtt beépítettem.
2. **Záró review** (Medium 2× → BLOCKING):
   - JSDoc a `buildExtensionRegistry`-ben pontatlan ("üres Map" vs. tényleges per-entry skip) → JSDoc frissítés két szint elkülönítésére.
   - Output normalizálás engedékenység (`{isValid:false, errors:[]}` és `{success:false}` `error` nélkül diagnosztika nélkül átengedett) → explicit `invalid_output_shape: ... empty` üzenet pótlás.
3. **Follow-up review** (BLOCKING marad): a validator `errors:[{code:'X'}]` (nem-string elemek) `isValid:true` mellett még csendben átment. → Strict shape-check helper-pár (`_checkValidatorShape`/`_checkCommandShape`) bevezetése, a normalizálás ELŐTT futtatva.
4. **Final pass review** (CLEAN): a strict shape-check lefedi az összes előző Medium és Low finding-et.

### Snapshot-immune szemantika

Az aktivált publikáció `compiledExtensionSnapshot`-ja a runtime kanonikus forrása. Az extension-doc Realtime mutáció (rename, code-szerkesztés, archive) NEM érinti az élő publikáció validálását vagy parancs-futtatását. Új aktiváció után új snapshot kerül be (a `activate_publication` action a workflow `compiled` JSON-ból frissen scan-eli az `ext.<slug>` hivatkozásokat és a hozzájuk tartozó extension-kódokat).
