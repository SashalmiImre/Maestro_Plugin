## Második session — A.4.9 review fix: Realtime Bus reconnect-time resync

Az A.4.9 review egy latent stale-forever kockázatra mutatott rá a `useContributorGroups` cache-en: a WS megszakadás-és-újrakapcsolódás ablakában érkezett szerver-mutációk nem érkeznek push-ként, és a Realtime-vezérelt cache-ek (memberships, aktív pub child, csoport listák) a következő mount-ig stale-ek maradnak. A Plugin-oldal a [[Döntések/0001-dual-proxy-failover|dual-proxy reconnect-réteg]]-gel védve van; a Dashboard közvetlenül beszél az Appwrite Cloud-dal, ezért szimmetrikusan saját reconnect-detektálás kell.

**SDK ellenőrzés**: a feladatleírás feltételezte, hogy a `client.subscribe()` reconnect promise-t ad — a 24.1.1 SDK valójában szinkron unsubscribe függvényt ad vissza. A subscribe callback CSAK `event` push-okat lát. Az SDK belső `connected` üzenete és a WS `open`/`close` event-ek nincsenek kitéve a publikus API-n. A `client.realtime.socket` az egyetlen alacsony szintű hookpoint — viszont minden reconnect-nél új instance.

**Implementáció** (`packages/maestro-dashboard/src/contexts/realtimeBus.js`):
- `subscribeRealtime(channels, callback, options)` kapott egy `options.onReconnect` callback opciót. Az unsubscribe-bal automatikusan eltávolítódik a reconnect-listenerek közül is.
- `installSocketHook()` egyszer (lazy, az első `doRebuild()`-nél) monkey-patch-eli a `client.realtime.createSocket()`-et: minden új socket-példányra `open`/`close` listener-t aggat. A MÁSODIK (vagy későbbi) sikeres `open` után tüzeli az `onReconnect` callback-eket — az ELSŐ `open` (kezdeti kapcsolódás) NEM, hogy a fogyasztó mount-effect fetch-jét ne duplikáljuk.
- Idempotens (csak az első futáskor patch-el), ref-deduped (no-op `createSocket()`-re nem aggatja újra a listener-eket), fail-safe (ha az SDK API eltűnik, warn + skip — a normál subscribe továbbra is működik).

**Fogyasztók wiring**:
- [AuthContext.jsx:431](packages/maestro-dashboard/src/contexts/AuthContext.jsx:431) — `silent: true` membership reload reconnect-kor (tranziens hiba ne ürítse a már érvényes scope-ot; destructive eseteket egy 403 vagy tényleges Realtime delete event jelzi).
- [DataContext.jsx:760](packages/maestro-dashboard/src/contexts/DataContext.jsx:760) — `resyncRealtimeData` useCallback: publikációk + aktív kiadvány child rekordok + workflow listák. Detektálja, ha az aktív kiadvány disconnect alatt törlődött, és a derived state-et clearel-i.
- [useTenantRealtimeRefresh.js:69](packages/maestro-dashboard/src/hooks/useTenantRealtimeRefresh.js:69) — debounce nélküli `reload()` reconnect-kor.
- `WorkflowDesignerPage.jsx` SZÁNDÉKOSAN KIMARAD — single-doc subscribe, a felhasználó nem mentett változtatásait nem akarjuk auto-resync-kel letörölni.

**Nyitva hagyott**: a `useContributorGroups` hook 5 perces module-szintű cache-t tart, de NEM iratkozik fel Realtime-ra (az A.4.9 review feltevése volt — valójában nincs ilyen kód a fájlban). Ha utólag bekerül a Realtime listening, az `onReconnect` opcióval invalidate + refetch-et kell beállítani. Addig a stale ablak a TTL (5 perc) vagy scope-váltás.

**Verifikáció**: Vite production build sikeres (312 modul, 797 KB). A reconnect viselkedést élesben nem teszteltem — manuális teszthez DevTools offline toggle + szerver-oldali mutáció a disconnect-ablakban kellene.

## Negyedik session — Harden pass az A.5 munkán

A `/harden` skill 7-fázisú pipeline-ja végigvitte az A.5 unstaged változtatásait (787+/79−, 6 kódfájl + 4 docs).

**Codex baseline review** 1 critical + 1 P1 + 1 P2 problémát talált:
- 🔴 **Cross-user permission leakage**: a `logout` / `sessionExpired` ágon nem bumpolt a `permissionsGenRef` (és a `groupSlugsGenRef` sem volt teljes védelemmel ellátva); a `refreshPermissions` setUser-callback nem ellenőrzött user-azonosságot. User A logout → User B login között in-flight permission lookup A office-szal visszaérkezhetett és A jogokat ír B-re.
- 🟠 **Tenant Realtime hiány**: az `organizationMemberships` és `editorialOfficeMemberships` változására a UserContext nem reagált, így egy Dashboard-oldali org-role promotion / office removal után a `permissions` snapshot stale maradt a következő scope-váltásig.
- 🟡 **`useContributorGroups` silent truncation**: 100 csoport / 500 tag limit cursor-pagination nélkül.

**Codex adversarial review** 7 design-pontot vetett fel — 5 NICE-TO-HAVE / DESIGN ACCEPTED, 1 megerősítette a baseline P2-t, 1 megerősítette a `useUserPermission` "deny on loading" kérdést (felhasználói döntéssel az A.7.2 alá került).

**Javítások** (4 db, MUST + SHOULD):
1. `bumpAllAuthGens()` Provider-szintű helper mind a 4 gen-counter-t bumpolja a login start / logout / sessionExpired ágakon. A `refreshGroupSlugs` és `refreshPermissions` setUser-callback-ekbe `prev.$id !== currentUserId` belt-and-suspenders guard.
2. Új `useEffect` a UserContext-ben: subscribe `organizationMemberships` + `editorialOfficeMemberships` (300ms debounce, userId-szűrt non-delete event-ekre) → `loadAndSetMemberships` + `refreshPermissions`.
3. `useContributorGroups` cursor-pagination 100/lap × 10-lap cap (1000 doc max) + cap-warning logger.
4. A `fetchMemberPermissionSlugs` 3 inline lapozó loop-ja és a `useContributorGroups.fetchAllPaginated` egy közös `paginateAll` shared util-ba (`core/utils/promiseUtils.js`) mergel.

**Simplify pass** 5 további javítást hozott:
- `useContributorGroups` modul-szintű `CACHE` invalidálódik `MaestroEvent.authStateChanged`-en (cross-user leak analóg).
- `paginateAll` shared util (a 4 inline loop egy implementációba mergelve).
- `useUserPermissions` `slugsKey = slugs.join('|')` deps-stabilizáció (inline literal slug-array nem triggerel feleslegesen).
- `_badge` UI-mező mostantól külön `badgesBySlug: Map<slug, badge>` a `ContributorsSection`-ben (nem mutáljuk a hook által visszaadott group objektumot).
- 3 WHAT-not-WHY komment dropolva, "Cross-user leakage" repetíció rövidítve.

**Verifikáló Codex review**: BLOCKER CLEAN. Két NICE-TO-HAVE megjegyzés design-elfogadott:
- A `loadAndSetMemberships` és `refreshPermissions` tranziens hibára state-megőrző viselkedése — fail-soft minta a recovery-flow-ra.
- `.delete` event-ek nem szűrnek `userId`-re — szándékos (Appwrite delete payload incomplete lehet).

**Codex stop-time review fix**: scope-váltáskor a `refreshGroupSlugs`/`refreshPermissions` hibakor megőrizte a `prev`-et — viszont az `prev` az **előző office** snapshot-ja, fail-open egy másik office-ra váltás közben. Megoldás: a `scopeChanged` handler azonnal nullázza a `groupSlugs`-ot és `permissions`-t a refresh-ek INDÍTÁSA ELŐTT (eager-clear); hibakor `null`/`[]` marad (konzervatív loading-state), nincs cross-office stale state.

**Új feladat**: [[Feladatok|A.7.2]] — Plugin `useUserPermission` "deny on loading" API revizit. Triggerelje az első valódi UI consumer.
