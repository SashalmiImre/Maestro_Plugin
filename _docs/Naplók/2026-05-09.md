### MCP-vel direkt schema-bootstrap workaround

A Codex tervezte `bootstrap_organization_status_schema` és `bootstrap_organization_invite_history_schema` action-ök `requireOwnerAnywhere(ctx)` guard-ot használnak, amely a `ctx.callerId`-t a `req.headers['x-appwrite-user-id']`-ből veszi. Az Appwrite REST API blokkolja az `x-appwrite-*` header-eket az `functions_create_execution` hívásnál, és az API key-vel nem lehet `caller user`-t impersonate-elni. **Workaround**: az MCP-vel direkt DB API-hívások (`tables_db_create_enum_column`, `tables_db_create_string_column`, `tables_db_create_index`) — ugyanaz az eredmény, NEM kell user-context. A CF action helye marad a kódban (ha valaki Console-ról futtatná).

### Harden Fázis 1-7 (1 commit-ban beolvasztva az F+E+G-vel)

A `/harden` skill 7-fázisú minőségi pass: 1 baseline + 1 adversarial + szintézis + javítások + simplify + verifikáló + összefoglalás.

#### Harden 4. fázis: javítások (3 MUST FIX + 1 SHOULD FIX)

- **`backfill_admin_team_acl` reconcile** (Baseline #1 + Adversarial BLOCKER 2): a tisztán additív backfill nem védte az out-of-band membership-változásokat (Console-ról törölt admin bent ragadt). Új: stale-tagok eltávolítása az admin-team-ből a `privilegedMemberships` halmazhoz viszonyítva.
- **`acceptInvite` admin-role: `ensureTeam` ELŐSZÖR + fail-closed** (Baseline #2 + #3): korábban best-effort volt; ha az admin-team nem létezett (legacy org backfill előtt), a `team_not_found` némán elnyelődött → admin user 'admin' role-t kapott, DE NEM látott ACL-t. Most: `ensureTeam` (idempotens, létrehozza) + `ensureTeamMembership`. Ha bármelyik bukik, fail-closed (a user retry-olhatja).
- **Frontend `swallow403` lambda** (Baseline #4): csak a 403-at fordítja üres listára; minden más hibát továbbdob → `loadError` szignál a teljes modal-on.
- **`change_organization_member_role` org-existence re-check + `adminTeamSyncSkipped` flag** (Adversarial DESIGN): a delete-race ellen org-doc gyors lookup; ha a `permissionContext.orgStatusByOrg` cache-ben már van, a redundáns getDocument SKIP. A response-ban `adminTeamSyncSkipped: true` flag — a frontend tájékoztathat a backfill-igényről.

#### Harden 5. fázis: 3 simplify FIX

- **Efficiency #1**: `change_organization_member_role` reuse `permissionContext.orgStatusByOrg` cache → -1 redundáns DB read minden role-change-en.
- **Efficiency #3**: `backfill_admin_team_acl` stale-removal `Promise.all` 10-es chunked parallelism → 10× speedup nagy legacy orgon.
- **Quality #1**: `_archiveAndUpdateExpiredInvite` wrapper a 4 opportunista expire-ágon (claim-then-update minta egy helyen).

#### Harden 6. fázis: verifikáló stop-time review

A 4+5. fázis változásokra **1 BLOCKER észlelve**: az `acceptInvite` admin-ágon a fail-closed admin-team-sync **részleges sikert** hagyott (membership létrejön, ezután 500 → user TAG, de NEM admin-team-tag). A retry az idempotens-ágra esett, ami nem javította.

**Fix**: idempotens admin-team-sync az `idempotent_already_member` és `race_winner_already_member` ágakon. Best-effort, log non-blocking — ha újra bukik, a következő retry vagy a `backfill_admin_team_acl` pótolja.
